## Installation Workflow

### Standard Path (New Machine)

```bash
# 1. Boot NixOS installer ISO
# 2. Partition disks (manually or with disko)
# 3. Mount filesystems to /mnt
# 4. Generate hardware config
nixos-generate-config --root /mnt

# 5. Edit /mnt/etc/nixos/configuration.nix
# 6. Install
nixos-install

# 7. Set root password (if not declared)
nixos-enter --root /mnt -c 'passwd root'

# 8. Reboot
reboot
```

### Remote Install with nixos-anywhere (No Physical Access)

Installs NixOS on any Linux machine over SSH using kexec. No ISO needed.

```bash
# Requirements: target ≥1 GB RAM, SSH root access, disko config in flake
nix run github:nix-community/nixos-anywhere -- \
  --flake .#myhostname \
  root@192.168.1.10
```

With custom disk config:
```bash
nix run github:nix-community/nixos-anywhere -- \
  --flake .#myhostname \
  --disk-encryption-keys /tmp/disk.key /dev/stdin \
  root@192.168.1.10
```

## Disk Partitioning with disko

disko declaratively handles partitioning, formatting, and mount point generation. Never write `fileSystems` manually when using disko.

### Minimal EFI + ext4

```nix
# disko-config.nix
{ disko.devices.disk.main = {
  device = "/dev/sda";   # or use /dev/disk/by-id/... for stability
  type = "disk";
  content = {
    type = "gpt";
    partitions = {
      ESP = {
        size = "512M";
        type = "EF00";
        content = {
          type = "filesystem";
          format = "vfat";
          mountpoint = "/boot";
          mountOptions = [ "umask=0077" ];
        };
      };
      root = {
        size = "100%";
        content = {
          type = "filesystem";
          format = "ext4";
          mountpoint = "/";
        };
      };
    };
  };
}; }
```

### LUKS Encryption + btrfs

```nix
{ disko.devices.disk.main = {
  device = "/dev/sda";
  type = "disk";
  content = {
    type = "gpt";
    partitions = {
      ESP = {
        size = "512M";
        type = "EF00";
        content = { type = "filesystem"; format = "vfat"; mountpoint = "/boot"; };
      };
      luks = {
        size = "100%";
        content = {
          type = "luks";
          name = "cryptroot";
          settings.allowDiscards = true;
          content = {
            type = "btrfs";
            extraArgs = [ "-L" "nixos" "-f" ];
            subvolumes = {
              "/root"   = { mountpoint = "/";     mountOptions = [ "compress=zstd" "noatime" ]; };
              "/home"   = { mountpoint = "/home";  mountOptions = [ "compress=zstd" "noatime" ]; };
              "/nix"    = { mountpoint = "/nix";   mountOptions = [ "compress=zstd" "noatime" ]; };
              "/persist"= { mountpoint = "/persist"; mountOptions = [ "compress=zstd" "noatime" ]; };
            };
          };
        };
      };
    };
  };
}; }
```

### Apply disko partitioning

```bash
# Wipes and partitions disk — destructive!
sudo nix run github:nix-community/disko -- \
  --mode disko \
  ./disko-config.nix

# Then generate config without filesystem declarations (disko handles them):
nixos-generate-config --no-filesystems --root /mnt
```

Include disko module in NixOS config so `fileSystems` is auto-generated:
```nix
imports = [ disko.nixosModules.disko ./disko-config.nix ];
```

## hardware-configuration.nix

**Generated by** `nixos-generate-config`. Contains machine-specific hardware settings.

```bash
# Generate (overwrites existing)
nixos-generate-config --root /mnt

# Regenerate on existing system after hardware change
sudo nixos-generate-config
```

**Contains:**
- `boot.initrd.availableKernelModules` — modules needed to mount root
- `boot.kernelModules` — always-loaded modules
- `fileSystems` and `swapDevices` (unless using disko)
- CPU microcode (`hardware.cpu.intel.updateMicrocode` / `amd.updateMicrocode`)
- `nixpkgs.hostPlatform`

**Rules:**
- Keep in version control — it is machine-specific
- Do NOT edit manually after generation (re-run generate-config instead)
- When using disko, generate with `--no-filesystems`

## nixos-hardware Profiles

Pre-built hardware profiles that fix quirks for specific machines.

```nix
# flake.nix input
inputs.nixos-hardware.url = "github:NixOS/nixos-hardware/master";

# In nixosSystem modules:
nixos-hardware.nixosModules.lenovo-thinkpad-x1-carbon-gen9
nixos-hardware.nixosModules.dell-xps-15-9500-intel
nixos-hardware.nixosModules.raspberry-pi-4
nixos-hardware.nixosModules.common-pc-ssd
nixos-hardware.nixosModules.common-cpu-amd-pstate
```

Browse available profiles: `github:NixOS/nixos-hardware`

## Boot Loader Configuration

### systemd-boot (EFI — recommended)

```nix
boot.loader.systemd-boot.enable = true;
boot.loader.systemd-boot.configurationLimit = 10;   # prevent /boot fill-up
boot.loader.efi.canTouchEfiVariables = true;
```

### GRUB (BIOS or EFI fallback)

```nix
boot.loader.grub = {
  enable = true;
  device = "/dev/sda";      # BIOS: disk, not partition
  # device = "nodev";       # EFI mode
  # efiSupport = true;      # EFI mode
  configurationLimit = 10;
};
```

## stateVersion — Critical Rule

**Set once at install time. Never change it.**

```nix
system.stateVersion = "26.05";   # set to NixOS version at time of install
```

- Controls migration behavior for stateful services (databases write paths, PostgreSQL data dirs, etc.)
- Does NOT control which packages are installed — your `nixpkgs` flake input does that
- Changing it across existing machines can corrupt service state or break upgrades
- Mismatched `stateVersion` across hosts is fine — each machine keeps its own value

## Flake-based Installation Config Skeleton

```nix
# flake.nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-26.05";
    # Variant: swap to the weekly-cooldown mirror for a 7-day supply chain buffer
    # nixpkgs.url = "https://flakehub.com/f/DeterminateSystems/nixpkgs-weekly/0.1";
    # See configuration.md "Variant: nixpkgs weekly cooldown" for details.
    disko = { url = "github:nix-community/disko"; inputs.nixpkgs.follows = "nixpkgs"; };
    nixos-hardware.url = "github:NixOS/nixos-hardware/master";
    agenix = { url = "github:ryantm/agenix"; inputs.nixpkgs.follows = "nixpkgs"; };
  };

  outputs = { self, nixpkgs, disko, nixos-hardware, agenix, ... }@inputs:
  {
    nixosConfigurations.myhostname = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux";
      specialArgs = { inherit inputs; };
      modules = [
        disko.nixosModules.disko
        ./disko-config.nix
        nixos-hardware.nixosModules.common-pc-ssd
        agenix.nixosModules.default
        ./hardware-configuration.nix
        ./configuration.nix
      ];
    };
  };
}
```
