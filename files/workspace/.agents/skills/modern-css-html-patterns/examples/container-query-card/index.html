Container query card

The outer frame is intentionally resizable on desktop so the component can prove that it responds to its own inline size.

Layout follows the card, not the viewport

The card switches from stacked to split when its own inline size crosses the query.

  • container-type: inline-size scopes the query to this card.
  • @container changes grid tracks only when the card is wide enough.
  • The fallback is the stacked card, which stays readable at every size.
